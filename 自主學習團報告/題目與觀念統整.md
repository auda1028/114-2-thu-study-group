# 競程題目與觀念統整

## 一、使用目的

本份統整將競程練習題整理成適合閱讀與複習的學習資料。內容重點不是列出所有程式檔，而是說明每一類題目背後對應的資料結構、演算法與解題思考方式，方便其他學習者依照主題循序複習。

整體成果包含 40 題解題報告、主題式 code book，以及 Ocean Cup / NCPC 實戰模擬整理。三者的關係如下：

| 資料 | 作用 | 適合閱讀方式 |
|---|---|---|
| 40 題解題報告 | 單題題意、程式與解法說明 | 遇到不熟的題型時逐題閱讀 |
| Code Book | 按主題整理常用模板與技巧 | 複習演算法與比賽前查閱 |
| 實戰模擬報告 | 將題型辨識用在比賽情境 | 學習讀題、分工、除錯與時間分配 |

| 主題 | 核心觀念 | 代表題型 | 學習目標 |

| 字串與頻率 | 線性掃描、字元計數、回文條件、map 統計 | Repetitions、Palindrome Reorder、Hardwood Species | 能把字串問題轉成頻率或狀態問題 |
| 排序與雙指針 | 排序後掃描、左右指針、保留索引 | Apartments、Ferris Wheel、Sum of Two Values | 能利用排序降低搜尋複雜度 |

## 四、觀念學習路線

1. 先熟悉 C++ 競程模板、輸入輸出、多測資與 `long long`。
2. 練習基礎模擬與數學規律題，建立把文字規則轉成程式的能力。
3. 進入排序、雙指針與貪心，學習如何用資料順序降低時間複雜度。
4. 學習前綴和、滑動視窗與掃描線，處理區間查詢與連續片段問題。
5. 補上 set、multiset、Fenwick Tree、stack、queue 等資料結構。
6. 最後整理 BFS、DFS、DSU、快速冪與大數，形成比賽可查閱的 code book。

## 五、實戰模擬的學習價值

Ocean Cup 與 NCPC 題目適合用來檢驗平時練習是否能轉換成實戰能力。正式比賽中，解題不只需要會寫單一演算法，也需要快速完成以下判斷：

| 比賽情境 | 需要的能力 |
|---|---|
| 開場掃題 | 判斷哪些題目可先拿分 |
| 多次錯誤提交 | 快速定位格式、邊界或陣列越界問題 |
| 題目看似模擬但資料量大 | 判斷是否需要預處理、資料結構或圖論建模 |
| 題目敘述複雜 | 先抽出狀態、操作與目標，再決定演算法 |
| 團隊合作 | 分工讀題、實作、測資檢查與除錯 |

## 六、成果使用方式

閱讀這份成果時，可以先從題解目錄選一題進入，再回到 code book 找相同主題的模板。例如讀到 Concert Tickets 時，可以對照 multiset 的前驅查詢；讀到 Dungeon Master 時，可以對照 BFS 與三維狀態設計。這樣能把「單題 AC」延伸成「可重複使用的解題方法」。
