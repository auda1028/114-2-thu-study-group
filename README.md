# 競程自主學習期末成果 README

這份資料夾是競程自主學習團的期末成果整理，內容包含學習總報告、主題式 code book，以及 40 題附註解程式碼的解題報告。閱讀對象設定為老師、同學與其他想了解這份學習成果的人，因此內容重點放在「看得懂、能學習、能複習」，不是單純保存原始程式檔。

| `自主學習團報告/codebook.md` | 按主題整理的競程 code book | 複習演算法、資料結構與常用模板 |

## 建議閱讀順序

1. 先讀 `自主學習團報告/題目與觀念統整.md`  
   先建立整體概念，了解題目被分成哪些主題，例如基礎模擬、數學規律、排序貪心、資料結構、圖論與數論。

2. 再讀 `自主學習團報告/codebook.md`  
   依章節複習常用技巧與模板，例如 stack、queue、prefix sum、sliding window、set、multiset、Fenwick Tree、BFS、快速冪等。

3. 接著打開 `題解題報告/README.md`  
   這裡是 40 題總目錄，可以依題目名稱或核心觀念找到對應的單題報告。

4. 最後閱讀單題題解  
   每一題都包含題目連結、題意摘要、參考程式與註解、解題觀念與複雜度，適合拿來當解題報告或複習教材。

## 40 題題解的格式

每份單題報告大致包含：

| 區塊 | 說明 |
|---|---|
| 題目 | 題目連結與題意摘要 |
| 參考程式與註解 | 可以閱讀的 C++ 解法，並在關鍵步驟加上註解 |
| 解題觀念 | 說明為什麼這樣解、使用哪些資料結構或演算法 |
| 複雜度 | 說明時間複雜度，幫助判斷解法是否符合題目限制 |

這些題解中的 code 不是只拿來提交，也可以用來學習「如何把題目轉成演算法」。

## 如何使用這份成果

### 給第一次閱讀的人

先看 `題目與觀念統整.md`，再看 `題解題報告/README.md`。這樣可以先理解整體學習方向，再挑選有興趣的題目閱讀。

### 給想學 code 的人

直接從 `題解題報告/README.md` 選題。每份題解中的「參考程式與註解」可以當作 code 範例，「解題觀念」則說明程式背後的演算法想法。

### 給想複習競程觀念的人

閱讀 `codebook.md`。它不是照題目順序排列，而是依照主題整理，適合在比賽前或練習前快速複習。

### 給要做成果報告的人

建議使用以下順序介紹：

1. 自主學習目標：透過競程題目練習資料結構與演算法。
2. 學習內容：40 題題解、主題式 code book、題型統整。
3. 學習方法：先解題，再整理 code，最後把題目歸類成可複習的觀念。
4. 學習成果：能從題目特徵判斷適合的演算法，並用 C++ 實作。

## 成果重點

- 不是單純堆放程式，而是把題目整理成可閱讀的學習資料。
- 每題都有題意、code、註解、解法與複雜度。
- Code book 以主題分章，方便查找與複習。
- 題目涵蓋 CSES、CPE/UVa 經典題與 BFS、Fenwick Tree、set/multiset、貪心、數論等競程核心觀念。

## 使用提醒

- 若只想快速了解成果，先看 `自主學習團報告/題目與觀念統整.md`。
- 若想查某一題的 code，先看 `題解題報告/README.md`。
- 若想複習某個演算法，直接看 `自主學習團報告/codebook.md`。
- 單題題解中的程式碼是學習用參考，閱讀時應搭配解題觀念理解，不建議只背程式。
